# Workspace Project Rules

<RULE[error_transparency_and_learning]>
Whenever a command, compilation, or runtime operation produces an error:
1. EXPLAIN THE ROOT CAUSE: State clearly what went wrong and why the error happened before or alongside applying the fix.
2. EXPLAIN THE SOLUTION: Detail how the code/command was modified to resolve it so the user gains insight and technical experience.
3. PRESERVE ACCOUNTABILITY: Never silently patch errors without acknowledging the diagnostic cause to the user.
</RULE[error_transparency_and_learning]>
